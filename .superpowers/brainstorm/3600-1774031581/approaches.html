Three approaches to the search feature

All three include the dial-up modem sound. The main difference is what happens when you click a search result link.

A

Search + dead links (simplest)

Groq generates 4–5 results rendered as a Yahoo directory results page. Links are visual only — clicking them does nothing (or shows a brief "Unable to connect" tooltip). Clean and fast to build.

Pros

  • Least code
  • No new page to design

Cons

  • Dead links feel incomplete
  • Breaks player immersion
B

Search + authentic 404 page (recommended)

Groq generates results. Clicking any result navigates to a period-accurate Netscape 404 error page ("The requested URL was not found on this server"). This is historically perfect — broken links were everywhere in 1995. Adds to the era's feel of fragility and impermanence.

Pros

  • Historically accurate
  • Links feel real (and appropriately dead)
  • Only one new page to build

Cons

  • Every result is a dead end
C

Search + smart results (richest)

Groq generates results, but the existing 4 pages (Yahoo, Yahoo Computers, Valley Computer, Vintage Archive) surface as top results for relevant queries. Those links navigate to the real pages. All other generated links go to the 404 page. Rewards players who search for things related to the level.

Pros

  • Some links actually work — feels most alive
  • Rewarding for explorers

Cons

  • Most complex to build
  • Groq prompt needs tuning to surface right pages
