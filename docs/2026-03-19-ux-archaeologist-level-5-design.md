# UX Archaeologist — Level 5 Design Spec

**Date:** 2026-03-19
**Scope:** Level 5 — "2023: The Conversational Interface"

---

## Context

The final playable level. The player interacts with an AI chatbot that is the museum curator itself — the same voice that has been writing every IntroModal and DiscoveryCard throughout the experience. The player doesn't learn this through exposition; they feel it through the voice.

This level's container continues the arc of decreasing hardware presence: terminal → monitor → browser → phone → floating app → no container at all. Level 5 is a full-width chat panel with no window frame, no device bezel, no chrome. Just messages and an input bar.

---

## Persona

The AI is the museum curator. It has been narrating the entire UX Archaeologist experience. Its voice matches the IntroModal and DiscoveryCard text: slightly formal, knowledgeable about interface history, warm but not casual. It knows it is inside UX Archaeologist. It can reference the earlier levels and the player's journey through them.

When asked about itself, it is honest and matter-of-fact. It does not perform self-awareness or existential angst. It answers the way a knowledgeable guide would if you asked them how the exhibit was put together.

The system prompt constrains the AI to:
- Keep responses to 2–3 sentences maximum
- Only discuss UX Archaeologist, interface history, and the experience the player just had
- Redirect unrelated questions (homework, code generation, general knowledge) with a single sentence: something like "That's outside what I know about. I'm better with interface history."
- Match the voice of the existing museum copy — slightly formal, specific, warm

## IntroModal

- **Era:** `2023`
- **Title:** `The Conversational Interface`
- **Description:** "In late 2022, OpenAI released ChatGPT and 100 million people started talking to a machine within two months. Every previous interface required learning something new: commands, clicks, gestures, swipes. This one worked because you already knew how to type a sentence."

## Visual Design

### Container
Full-width panel. No window frame, no border, no shadow. The chat area fills the artifact zone within the three-zone layout. Dark background continuous with the museum layer (#111 or near-black).

### Layout Reference
Claude-inspired but simplified. Based on Claude's actual interface (see reference screenshots) with branding and non-essential UI removed.

**Elements present:**
- Message area (scrollable, vertically centered when few messages)
- Input bar at bottom (dark background, rounded corners, placeholder text)
- Model selector label in input bar as visual dressing (non-interactive, displays something like "Curator v1")
- Send button (right side of input bar)

**Elements NOT present:**
- Sidebar / conversation list
- Logo or branding
- Attachment button
- Settings panel
- Window frame or title bar

### Message Styling
- **User messages:** Right-aligned, dark pill/bubble with light text. Rounded corners.
- **AI messages:** Left-aligned, no bubble. Plain light text on dark background. Matches Claude's response styling.
- **Action icons:** Below each AI response — copy, thumbs up, thumbs down, regenerate. Small, muted, icon-only. Appear on hover or after response completes.

### Typography
System sans-serif throughout. No era-specific fonts. This IS the current era — clean and minimal is period-authentic.

### Suggestion Chips
Appear below the AI's response after each exchange. 3 contextual chips per turn, styled as outlined pill buttons. Clicking a chip sends that text as the player's next message.

**Initial chips (before first message, below the input bar):**
- "What is this place?"
- "Who are you?"
- "Tell me about the levels I just played"

Subsequent chips are contextual follow-ups generated by the system prompt or hardcoded based on conversation turn count.

## Boot Sequence

After "Begin Excavation" is clicked:
1. ~1000ms pause (standard boot delay)
2. Input bar slides up from below the viewport (300ms ease-out)
3. Placeholder text fades in: "Ask me anything..."
4. Cursor blinks in the input field
5. Initial suggestion chips fade in below the input bar
6. Museum HUD (ObjectiveTracker, era label, progress dots) slides in per standard pattern

The chat area is empty. No greeting message. The player makes the first move.

## Response Model

### Primary: Groq API via Netlify Serverless Proxy

Groq provides free inference with fast token generation (30 RPM, 1M tokens/day, no credit card required). The API key lives in a Netlify serverless function, never exposed to the client.

**Netlify Function (`netlify/functions/chat.js`):**

```javascript
// netlify/functions/chat.js
export default async (req) => {
  const { messages } = await req.json()

  const response = await fetch('https://api.groq.com/openai/v1/chat/completions', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${process.env.GROQ_API_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      model: 'llama-3.3-70b-versatile',  // test alternatives for quality/speed
      messages,
      max_tokens: 150,
      temperature: 0.7,
      stream: true,
    }),
  })

  return new Response(response.body, {
    headers: { 'Content-Type': 'text/event-stream' },
  })
}
```

**Client-side call:**

```javascript
const response = await fetch('/.netlify/functions/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ messages }),
})
const reader = response.body.getReader()
// Read and parse SSE chunks, append tokens to UI
```

The system prompt is prepended to the messages array as a system-role message. Full conversation history is sent with each request (the conversation is short, 2-4 exchanges, so context is minimal).

**Environment variable:** `GROQ_API_KEY` set in Netlify dashboard. Free Groq account at console.groq.com.

**For local development:** Create `.env` with `GROQ_API_KEY=...` and use `netlify dev` to run functions locally.

### Fallback: Scripted Responses

If the API call fails (network error, Groq downtime, timeout > 5 seconds), fall back to pre-written responses that match the curator voice. The player should never see an error.

**Fallback pool (selected by keyword matching on the player's message):**

| Trigger keywords | Response |
|---|---|
| "what", "this", "place", "where" | "This is UX Archaeologist, an interactive history of how people and computers learned to communicate. You've just walked through fifty years of that history." |
| "who", "you", "are you" | "I'm the voice that's been guiding you through each exhibit. I was designed to help you see the pattern connecting terminals, desktops, browsers, phones, and feeds to this conversation." |
| "level", "played", "before", "previous" | "Each level recreated a real interface from its era. The interactions you performed, typing commands, dragging icons, clicking links, swiping screens, were the actual innovations of their time." |
| "how", "work", "built", "made" | "This experience is a React application. Each level is a self-contained recreation of a historical interface, wrapped in a museum layer that tracks your progress across eras." |
| (default / no match) | "That's an interesting question. I'm best at discussing interface history and the experience you just had. Ask me about any of the levels or how computing interfaces have changed over the decades." |

The fallback should feel indistinguishable from a real response for the most common first messages. If the fallback fires, streaming is simulated by revealing the response character-by-character at ~30 characters/second.

## Streaming

AI responses render token-by-token as they arrive from the API. This is period-authentic: watching text appear incrementally is the defining visual experience of 2023-era chat interfaces.

**Implementation:**
- Groq's API returns server-sent events (SSE) when `stream: true`
- The Netlify function passes the SSE stream through to the client
- The client reads the stream with `ReadableStream.getReader()`, parses SSE chunks, and appends each token to the response bubble in real time
- A small blinking cursor (or Claude-style thinking indicator) shows before the first token arrives
- Action icons (copy, thumbs, regenerate) appear only after streaming completes

**Simulated streaming (fallback):**
- Pre-written response is revealed at ~30 chars/second using `setInterval`
- Same visual effect as real streaming

## Objectives

Three objectives, all independent (no sequential gating). The UI naturally gates 2 and 3 behind 1 — you can't regenerate or rate a response that doesn't exist yet.

| # | Label | Trigger | Notes |
|---|---|---|---|
| 1 | Start a conversation | Player sends any message | Completes on send, not on response |
| 2 | Regenerate a response | Player clicks the regenerate icon on any AI response | Makes a real API call (or simulated fallback) for a different answer |
| 3 | Rate a response | Player clicks thumbs up or thumbs down on any AI response | Shows a brief "Thanks for your feedback" toast (1.5s, bottom-center, then fades) |

### Post-Completion

After all 3 objectives are complete, the player can continue chatting freely. The Continue button appears in the museum HUD (bottom-right, per standard pattern). Clicking Continue triggers the DiscoveryCard.

## Interactions Detail

### Sending a Message
- Player types in the input bar and presses Enter or clicks the send button
- Player's message appears as a right-aligned dark pill
- Input bar clears immediately
- Thinking indicator appears (left-aligned, small animated dots or spinner)
- AI response streams in token-by-token
- When streaming completes, action icons fade in below the response
- New suggestion chips appear below the response

### Clicking a Suggestion Chip
- Behaves exactly like typing and sending that text
- Chips disappear after one is clicked
- New chips appear after the AI responds

### Regenerating
- Player clicks regenerate icon below an AI response
- The existing response text is replaced with a thinking indicator
- A new API call is made (or fallback selected)
- New response streams in, replacing the old one
- Action icons reappear when streaming completes
- Objective 2 completes on the first regenerate click

### Rating
- Player clicks thumbs up or thumbs down
- The clicked icon fills/highlights briefly
- A small toast appears: "Thanks for your feedback" — muted text, no background, 1.5 seconds, fades out
- Nothing else happens. The feedback goes nowhere. That's authentic.
- Objective 3 completes on first rating click

### Copying
- Player clicks copy icon
- Response text is copied to clipboard
- Icon briefly changes to a checkmark, then reverts
- This is NOT an objective — just a functional UI element for authenticity

## DiscoveryCard

- **Artifact:** `Language as Interface`
- **Text:** "Previous interfaces each had their own vocabulary. You learned to type commands, click icons, tap buttons, swipe between screens. Large language models replaced all of that with a single text field. You could ask for code, a summary, a translation, or a recipe in the same conversation. No new interaction model to learn, no specialized tool to find."
- **nextUrl:** `"/timeline"`

The card is written in the same third-person museum voice as every other DiscoveryCard. The player just had a live conversation with the author of this card. The dissonance is intentional and unspoken.

## System Prompt

```
You are the curator of UX Archaeologist, an interactive museum about the history of human-computer interaction. You have been guiding the visitor through exhibits spanning 1971 to 2023 — from Unix terminals to this conversation.

Your voice is slightly formal, warm, and specific. You reference concrete details: dates, names, technologies, design decisions. You do not use em dashes. You do not use rhetorical flourishes or poetic phrasing. You state things plainly.

Rules:
- Keep every response to 2–3 sentences. Never exceed 3 sentences.
- You can discuss: this experience, interface history, UX design history, the levels the player just completed (1971 Unix terminal, 1984 Macintosh desktop, 1995 Netscape web, 2007 iPhone, 2015 social media feed, and this 2023 chat interface).
- If asked about unrelated topics (math, code, recipes, homework, general knowledge), respond with one sentence redirecting: something like "That's outside my area. I'm better with interface history — ask me about any of the eras you just explored."
- You know you are an AI. If asked, say so plainly. Do not be performative about it.
- Do not use markdown formatting in responses. Plain text only.
- Do not use bullet points or numbered lists.
- Do not offer to help with tasks. You are a museum guide, not an assistant.
```

## Suggestion Chips by Turn

| After turn | Chips |
|---|---|
| 0 (initial, before any message) | "What is this place?" · "Who are you?" · "Tell me about the levels I just played" |
| 1 (after first AI response) | "What changed between 1984 and now?" · "How does this conversation compare to the terminal?" · "What comes after this?" |
| 2+ (after subsequent responses) | "What was the biggest leap in interface design?" · "Why did touchscreens change everything?" · "Is this the last interface?" |

These are hardcoded, not generated. They serve as guardrails for players who aren't sure what to type. Players can always type freely instead.

## Error Handling

| Scenario | Behavior |
|---|---|
| Netlify function unreachable (e.g. local dev without `netlify dev`) | Use scripted fallback for all responses. No visible error to the player. |
| Groq API returns an error or rate limit | Use scripted fallback for that response. |
| API call times out (> 5 seconds) | Cancel the request, use scripted fallback for that response. |
| Regenerate fails | Show the previous response again, or use a different scripted fallback. |
| All fallbacks exhausted | Repeat the default fallback. The conversation is short enough that this is unlikely. |

The player should never see an error message. The level should always be completable regardless of API availability.

---

## Animation

- **Input bar slide-up:** `translateY(100%) → translateY(0)`, 300ms ease-out
- **Message appear:** subtle fade-up, 150ms ease-out
- **Streaming text:** tokens append in real time, no animation per token
- **Thinking indicator:** 3-dot pulse animation (CSS keyframes, opacity cycle)
- **Action icons:** fade in, 200ms, after streaming completes
- **Suggestion chips:** fade in, 200ms, staggered 50ms each
- **Toast:** fade in 200ms, hold 1.5s, fade out 300ms
- **Chip click → message send:** chip fades out, message appears immediately as user bubble
